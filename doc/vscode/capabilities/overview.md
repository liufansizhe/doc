# 扩展功能概述
Visual Studio Code 为扩展提供了许多扩展扩展其功能的方法。有时很难找到合适的贡献点和 VS Code API 来使用。本主题将扩展功能分为几类。每个类别描述：
- 您的扩展可以使用的一些功能
- 指向有关使用这些功能的更详细主题的链接
- 一些扩展想法

但是，我们也对扩展施加了限制，以确保 VS Code 的稳定性和性能。例如，扩展无法访问 VS Code UI 的 DOM。

## 一、常见功能
Common Capabilities 是您可以在任何扩展中使用的核心功能。   
其中一些功能包括：
- 注册命令、配置、键绑定或上下文菜单项。
- 存储工作区或全局数据。
- 显示通知消息。
- 使用 Quick Pick 收集用户输入。
- 打开系统文件选取器，让用户选择文件或文件夹。
- 使用 Progress API 指示长时间运行的作。
## 二、主题
主题控制 VS Code 的外观，包括编辑器中源代码的颜色和 VS Code UI 的颜色。如果您曾经想通过使 VS Code 具有不同深浅的绿色来使它看起来像是在编写 Matrix，或者只是想创建终极的、极简的灰度工作区，那么主题适合您。

扩展思路：
- 更改源代码的颜色。
- 更改 VS Code UI 的颜色。
- 将现有 TextMate 主题移植到 VS Code。
- 添加自定义文件图标。
## 三、声明性语言功能
声明性语言功能为编程语言添加了基本文本编辑支持，例如方括号匹配、自动缩进和语法突出显示。这是以声明方式完成的，无需编写任何代码。有关更高级的语言功能（如 IntelliSense 或调试），请参阅编程语言功能。

扩展思路：
- 将常见的 JavaScript 代码片段捆绑到扩展中。
- 向 VS Code 介绍一种新的编程语言。
- 添加或替换编程语言的语法。
- 使用语法注入扩展现有语法。
- 将现有 TextMate 语法移植到 VS Code。
## 四、编程语言功能
编程语言功能添加了丰富的编程语言支持，例如 Hovers、Go to Definition、诊断错误、IntelliSense 和 CodeLens。这些语言功能通过 vscode.languages.* API 公开。扩展可以直接使用这些 API，也可以编写语言服务器并使用 VS Code 语言服务器库将其适应 VS Code。

尽管我们提供了语言功能及其预期用途的列表，但没有什么能阻止您创造性地使用这些 API。例如，CodeLens 和 Hovers 是内联显示其他信息的好方法，而诊断错误可用于突出显示拼写或代码样式错误。

扩展思路：
- 添加显示 API 示例用法的悬停。
- 使用诊断报告源代码中的拼写或 linter 错误。
- 为 HTML 注册新的代码格式化程序。
- 提供丰富的上下文感知 IntelliSense。
- 为语言添加折叠、痕迹导航和大纲支持。

## 五、Workbench 扩展
Workbench 扩展扩展了 VS Code Workbench UI。向文件资源管理器添加新的右键单击作，甚至可以使用 VS Code 的 TreeView API 构建自定义资源管理器。如果您的扩展需要完全自定义的用户界面，请使用 Webview API 使用标准 HTML、CSS 和 JavaScript 构建您自己的文档预览或 UI。

扩展思路：
- 将自定义上下文菜单作添加到文件资源管理器。
- 在 Side Bar 中创建一个新的交互式 TreeView。
- 定义新的 Activity Bar 视图。
- 在 Status Bar 中显示新信息。
- 使用 WebView API 呈现自定义内容。
- 贡献源代码控制提供程序。
